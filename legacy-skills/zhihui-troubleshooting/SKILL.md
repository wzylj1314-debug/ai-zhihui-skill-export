# AI智绘效果排查

Use this skill to diagnose generation or quality issues before escalating.

## Workflow

1. Read `references/troubleshooting-general.md` for broad failure handling.
2. Read `references/troubleshooting-by-feature.md` for feature-specific known issues.
3. Provide likely cause, immediate retry advice, and what information to collect if escalation becomes necessary.
4. Use `zhihui-risk-handoff` only after repeated failure, customer complaint, important delivery, suspected system issue, or account/backend uncertainty.

## Output

Do not escalate just because the user says failure, abnormal, distorted, or inaccurate. First give practical steps.
